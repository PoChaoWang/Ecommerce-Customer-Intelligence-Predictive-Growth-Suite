# Ecommerce Customer Intelligence & Predictive Growth Suite

這是一個專為電商環境設計的進階 MarTech 資料科學專案。透過整合 **RFM 價值模型**、**NLP 情緒分析**、**隨機森林 LTV 預測模型** 以及 **同期群與關聯分析**，旨在將原始數據轉化為可立即執行的行銷自動化決策與深度商業洞察。

## 🚀 專案介紹
在現代電商營運中，數據的價值不在於「存儲」，而在於「轉化」。本專案模擬資深資料科學家的工作流，建立了一套自動化的數據處理與建模系統。它不僅能回顧過去（客戶分群），還能預測未來（LTV 預測），並針對不同的業務場景（如流失挽回、新客擴張、產品診斷）產出具體的行動名單，縮短從數據洞察到行銷執行的距離。

## 🛠 模組功能說明

### 1. `phase1_rfm_sentiment.py` (價值與情緒標籤)
*   **RFM 核心計算**：計算用戶的 Recency (近期性)、Frequency (頻率) 與 Monetary (貢獻金額)，並進行標準化分組。
*   **情緒分析 (VADER)**：運用自然語言處理技術掃描評論，識別用戶情緒分值。
*   **客戶分群邏輯**：整合價值與情緒指標，將用戶自動歸類為 VVIP、近期新客、沉睡高價值客、流失預警客等。

### 2. `phase2_behavior_strategy.py` (行為特徵與策略名單)
*   **行為聚合**：將瀏覽與加購等事件數據轉化為可供模型訓練的特徵。
*   **產品健康診斷**：提取產品負評關鍵字並監控評分波動。
*   **行動名單生成**：實作 Lookalike 種子名單、流失挽回名單、以及包含具體商品 ID 的購物車放棄名單。
*   **品牌聲譽監控**：自動化品類情緒監控與風險等級標註。

### 3. `phase3_ltv_model.py` (預測性建模與 ML Ops)
*   **預測性 LTV 模型**：實作 90 天時間維度切割，使用隨機森林預測用戶未來的獲利貢獻。
*   **模型解釋 (SHAP)**：為每筆預測提供推力與阻力解釋，讓預測結果具備可理解性。
*   **模型效能監控**：建立再訓練機制，自動記錄 RMSE 並在效能退化時發出警告。

### 4. `phase4_cohort_analysis.py` (生命週期與關聯分析)
*   **同期群分析 (Cohort Analysis)**：計算各月份新客的留存率與 ARPU，評估客戶獲取質量。
*   **產品關聯分析 (Affinity Analysis)**：透過市場籃子分析找出「買了 A 也會買 B」的強關聯組合。

### 5. `main.py` (自動化工作流調度)
*   負責 Raw Data 的讀取與清洗。
*   串聯 Phase 1 至 Phase 4 的執行順序與數據流。
*   彙整所有分析結果並導出至 `schema/` 目錄。

## 📊 CSV 產出物目的與用途說明

本專案生成的檔案構成了一個完整的行銷決策體系：

### 1. 行銷自動化名單 (Marketing Action Lists)
*   **`Cart_Abandonment_List.csv` (購物車放棄名單)**
    *   **目的**：鎖定「有動機但未轉換」的用戶。
    *   **用途**：透過 `abandoned_product_ids` 發送精準的 EDM 或 App 推播，並結合 predicted_profit_90_days 決定是否給予折扣（高價值者可給予誘因）。
*   **`Lookalike_Seed_List.csv` (高價值種子名單)**
    *   **目的**：擴大客群基數。
    *   **用途**：上傳至 Facebook/Google 廣告後台作為 Seed Audience，利用平台演算法尋找與這些 VVIP 特徵相似的新客。
*   **`Churn_Risk_List.csv` (流失預警名單)**
    *   **目的**：挽回即將離開的高價值回購客。
    *   **用途**：針對 `primary_barrier` 發送專屬回娘家優惠，搶在用戶徹底流失前介入。
*   **`EDM_Suggestions.csv` (沉睡客選品建議)**
    *   **目的**：重新激活沉睡用戶。
    *   **用途**：根據該群體過去最愛的品類（Top 10）進行主題式行銷，降低電子報開信率低迷的問題。

### 2. 產品與品牌診斷 (Product & Brand Insights)
*   **`Product_Health_Report.csv` (產品健康度報告)**
    *   **目的**：監控個別產品品質。
    *   **用途**：找出 `top_pain_points` 並反饋給供應鏈或研發部門優化。
*   **`Sleepy_Product_Health_Report.csv` (沉睡原因診斷)**
    *   **目的**：釐清客戶流失的真因。
    *   **用途**：區分是「產品體驗差」還是「品牌遺忘」。前者需產品團隊檢討，後者則交由行銷重定向。
*   **`Brand_Health_Monitor.csv` (品牌聲譽監控)**
    *   **目的**：品類整體的公關預警。
    *   **用途**：透過 7 天/30 天的滾動情緒分析，第一時間發現某品類是否爆發系統性客訴（如：某品牌的電子產品批次故障）。

### 3. 進階策略分析 (Strategic Analysis)
*   **`Cohort_Retention.csv` & `Cohort_Revenue.csv` (同期群分析)**
    *   **目的**：評估客戶生命週期（LTV）健康度。
    *   **用途**：觀察不同月份進來的新客，哪一梯次的留存率最高？消費力最強？藉此驗證過去特定月份的行銷專案是否帶來了高品質用戶。
*   **`Product_Affinity.csv` (產品關聯分析)**
    *   **目的**：發現購物組合規律。
    *   **用途**：用於官方網站的「加購推薦」邏輯，或設計組合包（Bundle Sales）提高客單價（AOV）。
*   **`C360_Table.csv` (客戶 360 全貌表)**
    *   **目的**：建立單一事實來源。
    *   **用途**：供 BI 工具對接，實現全方位營運監控。

### 4. 模型治理 (Model Governance)
*   **`Model_Explanations_Table.csv` (模型解釋表)**
    *   **目的**：提高模型透明度。
    *   **用途**：告訴一線行銷人員「為什麼模型認為這個人有潛力」，例如：因為他最近瀏覽次數激增。
*   **`Model_Run_Metadata.json` (模型運行元數據)**
    *   **目的**：監控 AI 穩定性。
    *   **用途**：自動記錄 RMSE 誤差，預防模型過期（Model Drift），確保營運團隊使用的預測數據始終精準。

## 📂 資料來源與數據備註

### 資料來源
本專案使用的原始數據集來自於 Kaggle：[E-commerce Dataset](https://www.kaggle.com/datasets/abhayayare/e-commerce-dataset)。

### 數據一致性說明 (Data Consistency Note)
*   **假資料特性**：由於該數據集包含由 Python 腳本生成的合成數據 (Synthetic Data)，在某些情況下 `events.csv` 與 `orders.csv` 之間的時間戳記 (Timestamps) 可能會出現邏輯上的不一致（例如：加購行為晚於下單時間）。
*   **基準定義**：在本專案的所有分析流程（包括 RFM 分析與時間窗格切割）中，**一律以 `orders.csv` 的時間資料作為絕對基準**。
